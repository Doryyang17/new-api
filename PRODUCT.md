# Product

## Register

product

## Users

The primary users are administrators and operators who manage AI API gateway routing, billing, user access, request safety controls, and operational logs. They work in an authenticated dashboard and need fast diagnosis, predictable controls, and evidence-rich views.

## Product Purpose

This product provides a unified AI API gateway with management surfaces for channels, models, users, tokens, billing, and site-level safety controls. Success means operators can configure policy, understand why requests were accepted or blocked, and act without inspecting raw databases or server logs.

## Brand Personality

Pragmatic, technical, restrained. The interface should feel like an operations console: direct labels, compact data, clear status, and low decorative noise.

## Anti-references

Avoid marketing-page composition, oversized hero sections, decorative cards, unclear status badges, and tables that hide critical diagnostic fields behind vague summaries.

## Design Principles

- Put operational evidence close to the decision it explains.
- Prefer dense but scannable layouts over decorative whitespace.
- Make failure states diagnosable from the UI without requiring shell or database access.
- Keep controls familiar and reversible for administrative workflows.
- Preserve Chinese-first product copy while keeping protocol names and API terms literal.

## Accessibility & Inclusion

Target WCAG 2.1 AA contrast for text and controls. Preserve keyboard navigation and visible focus states for tables, filters, dialogs, and destructive actions. Do not use color as the only indicator for blocked, warning, or monitoring states.
