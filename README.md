### 系统可用时间

新增「系统可用时间」功能，用于配置服务不可用时间段。当前默认目标场景为 `22:00-08:00` 宵禁窗口。

功能行为：

- 管理后台新增「定制化功能」菜单，并在该菜单下提供「系统可用时间」配置。
- 支持配置是否启用、不可用开始时间、不可用结束时间、判定时区和提示文案。
- 宵禁状态下，主页展示替换为宵禁提示页，导航栏和站内其他页面仍保持可访问。
- 宵禁状态下，模型调用类 API 直接返回 `503` 和 `system_curfew` 错误，不再进入渠道分发，也不会调用上游渠道 API。
- 宵禁拒绝会写入错误日志，保留请求路径、时间窗口、时区、Token 上下文等审计信息。
- 同步覆盖新版前端和经典前端，并补充浅色、深色主题下的主页展示。

### 每日系统用量上限

新增「每日系统用量上限」功能，用于给整站配置每天可消耗的总额度上限。

功能行为：

- 管理后台「定制化功能」下新增每日系统用量配置，支持开关、每日额度上限和提示文案。
- 模型调用类 API 在进入渠道分发前检查当日系统用量，超过限制时直接拒绝请求，不再调用上游渠道 API。
- 用量统计跟随消费日志汇总，并提供 5 分钟级别的缓存快照，兼顾严格拦截与管理页展示性能。
- 排行榜页面展示系统当日用量概览，方便运营侧观察已用额度、剩余额度和限制状态。
- 限制检查失败时采用保守策略，避免统计异常时绕过整站额度上限。

### 登录注册安全改造

新增登录与注册安全改造，用于降低自动化注册、无效注册码撞库和 OAuth 自动建号风险。

功能行为：

- 接入 Cloudflare Turnstile 免费验证方案，登录、密码注册和 OAuth 首次注册补码等关键入口使用严格校验。
- 前端在 Turnstile 未加载、加载超时、验证失败或未完成时给出明确提示，避免用户只看到按钮不可用。
- 新增「注册码注册」开关；开启后，密码注册、OAuth 首次建号和 WeChat 首次建号都必须提供有效注册码。
- 注册码支持后台批量生成，格式固定为 20 位 `A-Z0-9` 随机字符；单个注册码只能被一个用户消费一次。
- 注册码管理页支持查看未使用和已使用清单、展示数量、全量复制、10 条分页；已使用清单展示 `user_id` 与用户名。
- OAuth 已有绑定账号保持原有直接登录；首次 OAuth 登录完成授权后进入专门的注册码补充页，补码成功后才创建账号。
- OAuth 待注册资料使用服务端 pending ticket 与过期时间保存，不把完整 OAuth 资料写入浏览器 cookie session。
- 无效注册码失败会进入轻量风控：连续 3 次失败临时封禁 1 小时，24 小时内累计 10 次失败封禁 24 小时；风控维度使用 HMAC 后的 IP 与浏览器指纹，支持 Redis，Redis 不可用时回退到内存。
- 当前改造不依赖邮件系统；邮件验证仍保持原有独立开关和流程。

### 提示词过滤与二次审核

新增提示词过滤系统，用于在请求进入上游渠道前完成本地合规检查和可选二次审核。

功能行为：

- 模型调用类 API 在渠道分发前读取请求体并执行提示词过滤，拦截命中规则的请求，不再调用上游渠道 API。
- 支持 `block`、`warn`、`monitor` 三种处理模式；`block` 返回可配置 4xx 状态码和稳定错误码，`warn` / `monitor` 通过响应头标记命中结果并继续请求。
- 支持内置规则、自定义正则规则、敏感词、分组白名单、渠道白名单、扫描长度上限、阻断提示文案和命中日志开关。
- 命中日志会保存脱敏后的预览、规则命中、分数、动作、请求路径和 Token 上下文，便于后续审计。
- 可配置兼容 moderation 接口的二次审核服务，支持自定义 base URL、模型、超时时间和 API Key；二次审核默认 fail-closed，审核失败时保持保守拦截。
- 新版前端在「安全设置 / 敏感词」页面提供总览、触发日志、规则集、自定义规则和词库文件管理入口。

### 提示词过滤词库管理

新增可运营的提示词过滤词库能力，用于维护大量敏感词和启用预置词库。

功能行为：

- 支持上传 `.txt` 词库文件，按文件内容计算 SHA-256，生成稳定词库 ID，并限制单文件大小与最大词条数。
- 支持词库启用/停用、名称、分类、权重和严格命中标记调整。
- 支持预览词库内容、在线更新词条、删除自定义上传词库。
- 内置 Konsheng Sensitive-lexicon 预置词库，来源和 MIT 许可证随仓库保留；预置词库默认禁用，管理员可按需预览、编辑和启用副本。
- 词库命中统一进入提示词过滤评分与日志链路，和内置规则、自定义规则共用同一套阻断/告警/监控策略。

### 中文化维护策略

本 fork 将前后端文案维护范围收敛为中文，减少多语言同步成本。

功能行为：

- 后端保留 `go-i18n` 兼容层，但仅维护 `zh-CN` 后端文案。
- default 前端保留 `i18next` / `react-i18next` 兼容层，但仅维护中文 locale。
- 移除 default 前端语言切换、多语言静态 key 同步脚本和非中文 locale 维护面。
- classic 前端同步移除语言切换入口和非中文 locale 文件，保留中文显示能力。

### 项目内置自定义徽标

新增项目内置自定义徽标资源，方便在不依赖外部域名的情况下替换站点 Logo。

功能行为：

- 将自定义透明 PNG 徽标分别打包到 default 与 classic 前端 `public` 目录。
- default 前端系统设置的 Logo 字段允许填写完整 URL，或填写以单个 `/` 开头的站内 public 路径。
- 管理后台可直接将 Logo 配置为 `/custom-logo.png`，前端会将其作为站点徽标并同步到浏览器 favicon。
- 保留默认 `logo.png`，便于未配置自定义 Logo 的部署继续使用原默认资源。

## 保留能力

除上述定制化功能外，项目继续继承 New API 原有能力，包括多模型网关、渠道管理、用户与令牌管理、用量日志、计费配置、模型广场、排行榜、支付集成和前后端管理控制台等功能。本 fork 当前以中文界面和中文文案维护为主。

## 上游同步说明

本仓库后续二开应继续以 `upstream` 作为上游来源，以 `origin` 作为个人 fork 发布目标。合并上游更新时，应先确认定制化功能没有被覆盖，并重点检查以下入口：

- 系统可用时间：后端强制拦截、前端主页替换、管理后台配置入口。
- 每日系统用量上限：调用前拦截、消费日志统计、排行榜用量概览。
- 登录注册安全改造：Turnstile 严格校验、注册码消费事务、OAuth pending ticket、WeChat 首次建号拦截和注册码风控。
- 提示词过滤：调用前过滤中间件、规则评分、二次审核、命中日志和管理后台敏感词页面。
- 提示词过滤词库：上传词库、预置 Konsheng 词库、词条预览/编辑和 `prompt_filter_setting.lexicon_files` 配置。
- 中文化维护：后端 `zh-CN` locale、default 前端中文 locale、classic 前端中文显示与语言切换移除状态。
- 自定义徽标：default/classic `public/custom-logo.png`、Logo 站内路径校验和 favicon 同步逻辑。
