## 定制化功能

### 系统可用时间

新增「系统可用时间」功能，用于配置服务不可用时间段。当前默认目标场景为 `22:00-08:00` 宵禁窗口。

功能行为：

- 管理后台新增「定制化功能」菜单，并在该菜单下提供「系统可用时间」配置。
- 支持配置是否启用、不可用开始时间、不可用结束时间、判定时区和提示文案。
- 宵禁状态下，主页展示替换为宵禁提示页，导航栏和站内其他页面仍保持可访问。
- 宵禁状态下，模型调用类 API 直接返回终止型 `403` 和 `system_curfew` 错误，不再进入渠道分发，也不会调用上游渠道 API；避免 Codex 等按 `5xx` 规则重试的 Agent 将其当作临时服务故障。
- 宵禁拒绝会写入错误日志，保留请求路径、时间窗口、时区、Token 上下文等审计信息。
- 触发日志支持回查关联用户，便于运营侧定位受影响账号。
- 同步覆盖 default 前端和 classic 前端，并补充浅色、深色主题下的主页展示。

### 每日系统用量上限

新增「每日系统用量上限」功能，用于给整站或指定模型配置每天可消耗的额度上限。

功能行为：

- 管理后台「定制化功能」下新增每日系统用量配置，支持全站开关、全站额度、多个模型独立额度和提示文案。
- 模型调用类 API 在进入渠道分发前检查当日系统用量；全站超限时拒绝所有模型，模型独立额度超限时只拒绝对应模型，不再调用上游渠道 API。
- 用量统计以消费日志为最终来源，按规范化模型名聚合，并提供 5 分钟级别的缓存快照，兼顾严格拦截与管理页展示性能。
- 排行榜页面展示全站及模型独立额度的当日用量概览，方便运营侧观察已用额度、剩余额度和限制状态。
- 限制检查失败时采用保守策略，避免统计异常时绕过整站额度上限。

### 登录注册安全改造

新增登录与注册安全改造，用于降低自动化注册、无效注册码撞库和 OAuth 自动建号风险。

功能行为：

- 接入 Cloudflare Turnstile 免费验证方案，登录、密码注册和 OAuth 首次注册补码等关键入口使用严格校验。
- 前端在 Turnstile 未加载、加载超时、验证失败或未完成时给出明确提示，避免用户只看到按钮不可用。
- 新增「注册码注册」开关；开启后，密码注册、OAuth 首次建号和 WeChat 首次建号都必须提供有效注册码。
- 注册码支持后台批量生成，格式固定为 20 位 `A-Z0-9` 随机字符；单个注册码只能被一个用户消费一次。
- 注册码管理页支持查看未使用和已使用清单、展示数量、全量复制、10 条分页；已使用清单展示 `user_id` 与用户名。
- OAuth 已有绑定账号保持原有直接登录；首次 OAuth 登录完成授权后进入专门的注册码补充页，补码成功后才创建账号。
- OAuth 待注册资料使用服务端 pending ticket 与过期时间保存，不把完整 OAuth 资料写入浏览器 cookie session。
- 无效注册码失败会进入轻量风控：连续 3 次失败临时封禁 1 小时，24 小时内累计 10 次失败封禁 24 小时；风控维度使用 HMAC 后的 IP 与浏览器指纹，支持 Redis，Redis 不可用时回退到内存。
- 当前改造不依赖邮件系统；邮件验证仍保持原有独立开关和流程。

### 提示词过滤与二次审核

新增提示词过滤系统，用于在请求进入上游渠道前完成本地合规检查和可选二次审核。

功能行为：

- 模型调用类 API 在渠道分发前读取请求体并执行提示词过滤，拦截命中规则的请求，不再调用上游渠道 API。
- 支持 `block`、`warn`、`monitor` 三种处理模式；`block` 返回可配置 4xx 状态码和稳定错误码，`warn` / `monitor` 通过响应头标记命中结果并继续请求。
- 支持内置规则、自定义正则规则、敏感词、分组白名单、渠道白名单、扫描长度上限、阻断提示文案和命中日志开关。
- 超长 Prompt 按头尾片段扫描，避免处理大型类二进制载荷时拖慢请求链路。
- 命中日志会保存脱敏后的预览、规则命中、命中词、分数、动作、请求路径和 Token 上下文，便于后续审计。
- 管理员排查入口可查看、复制和下载管理员专用完整文本；普通用户日志接口继续移除完整文本和 `admin_info`，避免敏感内容越权暴露。
- 提示词过滤返回 460 等拦截响应后，前端仍能保留并展示本次请求历史，避免用户只看到空状态。
- 日志详情视图支持查看提示词过滤的规则、动作、命中词和上下文摘要。
- 可配置兼容 moderation 接口的二次审核服务，支持自定义 base URL、模型、超时时间和 API Key；二次审核默认 fail-closed，审核失败时保持保守拦截。
- default 前端在「安全设置 / 敏感词」页面提供总览、触发日志、规则集、自定义规则和词库文件管理入口。

### 提示词过滤词库管理

新增可运营的提示词过滤词库能力，用于维护大量敏感词和启用预置词库。

功能行为：

- 支持上传 `.txt` 词库文件，按文件内容计算 SHA-256，生成稳定词库 ID，并限制单文件大小与最大词条数。
- 支持词库启用/停用、名称、分类、权重和严格命中标记调整。
- 支持预览词库内容、在线更新词条、删除自定义上传词库。
- 内置 3 个精简预置词库（涉政、色情、暴力），由 Konsheng Sensitive-lexicon 来源素材审计合并而来；旧全量 Konsheng 文件已从预置词库中移除，不再随运行时预设提供或生效。
- 预置词库默认禁用，管理员可按需预览、编辑和启用副本。
- 词库命中统一进入提示词过滤评分与日志链路，和内置规则、自定义规则共用同一套阻断、告警、监控策略。

### 充值额度统计与结算安全

新增用户累计充值额度统计，并统一加固不同支付渠道的到账结算流程。

功能行为：

- 充值订单新增冻结后的到账额度字段，在创建外部支付订单前完成额度换算和边界校验，避免付款后因配置变化或数值异常导致到账不一致。
- 易支付、Stripe、Creem、Waffo、Waffo Pancake 和管理员补单共用事务化结算逻辑，重复回调不会重复增加用户余额。
- 启动时为历史成功订单补齐可确认的到账额度，无法安全换算的异常记录保留后台告警，不会静默写入错误额度。
- 管理员用户详情新增「累计充值额度」，与当前余额、已用额度分开显示，方便运营核对用户真实充值贡献。
- 额度换算继续使用统一的饱和保护，异常超大值会被拒绝并记录，而不是发生整数溢出或负向记账。

### 批量测活与并发防护

新增轻量级请求行为检测，用于降低模型批量测活、快速轮询和无意义扫描对渠道及系统资源的消耗。

功能行为：

- 根据用户、Token 和可信客户端 IP 的短时请求量，结合重复内容、模型快速轮询、失败突发及失败后快速重试进行风险评分。
- 对 `hi`、`hello`、`ping`、`test`、`你是谁` 等精确无意义短文本只增加单项风险分，不会仅凭一次正常短问题直接拦截。
- 支持观察和拦截两种模式；观察模式只记录管理员诊断日志，拦截模式对中高风险请求返回 `429` 和 `Retry-After`。
- 风险处置只使用短时冷却及用户、Token、IP 临时限制，不执行永久封禁；已限制请求会在读取大请求体前直接拒绝。
- 增加用户级和 Token 级在途并发上限，覆盖普通、流式和 Realtime 请求；超限不排队，Redis 环境支持多节点统一计数和长连接租约续期。
- 管理后台「安全与限制 / 批量测活与并发防护」提供开关、模式、冷却时间、并发上限、日志开关和分组白名单配置；功能默认关闭，建议先观察后拦截。
- 基于 IP 的检测只有在显式配置 `TRUSTED_PROXIES` 后才启用，避免伪造转发头造成误封；多节点部署建议配置 Redis。

### 中文化维护策略

本 fork 将前后端文案维护范围收敛为中文，减少多语言同步成本。

功能行为：

- 后端保留 `go-i18n` 兼容层，但仅维护 `zh-CN` 后端文案。
- default 前端保留 `i18next` / `react-i18next` 兼容层，但仅维护中文 locale。
- default 前端不维护语言切换、多语言静态 key 同步脚本和非中文 locale。
- classic 前端同步移除语言切换入口和非中文 locale 文件，保留中文显示能力。
- 合并上游多语言改动时，默认保留本 fork 中文单语策略，不恢复多语言入口。

### 项目内置自定义徽标

新增项目内置自定义徽标资源，方便在不依赖外部域名的情况下替换站点 Logo。

功能行为：

- 将自定义透明 PNG 徽标分别打包到 default 与 classic 前端 `public` 目录。
- default 前端系统设置的 Logo 字段允许填写完整 URL，或填写以单个 `/` 开头的站内 public 路径。
- 管理后台可直接将 Logo 配置为 `/custom-logo.png`，前端会将其作为站点徽标并同步到浏览器 favicon。
- 保留默认 `logo.png`，便于未配置自定义 Logo 的部署继续使用原默认资源。

### 前端构建与展示修复

围绕二开功能和上游合并，本 fork 额外维护了若干前端稳定性修复。

功能行为：

- classic 前端 Docker 构建会显式定位 Semi 依赖中的 `date-fns`，避免生产构建找不到日期依赖。
- default 前端统一额度展示单位，减少美元额度、Tokens 和配额展示之间的混淆。
- 认证、提示词设置、系统信息、分组倍率等页面在上游合并后保持中文单语和本 fork 交互规则。

## 保留能力

除上述定制化功能外，项目继续继承 New API 原有能力，包括多模型网关、渠道管理、用户与令牌管理、用量日志、计费配置、模型广场、排行榜、支付集成和前后端管理控制台等功能。本 fork 当前以中文界面和中文文案维护为主。

## 上游同步说明

本仓库后续二开应继续以 `upstream` 作为上游来源，以 `origin` 作为个人 fork 发布目标。合并上游更新时，应先确认定制化功能没有被覆盖，并重点检查以下入口：

日常二开以 `main` 作为唯一长期维护分支，验收后直接提交并推送 `origin/main`。临时开发或上游合并保护分支只用于短期回退，确认代码已进入 `main` 后应及时删除。

- 系统可用时间：后端强制拦截、前端主页替换、管理后台配置入口、宵禁日志用户展示。
- 每日系统用量上限：全站与模型独立额度、调用前拦截、模型名规范化统计、排行榜用量概览。
- 登录注册安全改造：Turnstile 严格校验、注册码消费事务、OAuth pending ticket、WeChat 首次建号拦截和注册码风控。
- 提示词过滤：调用前过滤中间件、规则评分、二次审核、命中日志、460 后历史保留、日志详情和管理后台敏感词页面。
- 提示词过滤词库：上传词库、3 个精简预置词库、词条预览/编辑和 `prompt_filter_setting.lexicon_files` 配置。
- 充值额度：订单到账额度冻结、历史订单补齐、各支付渠道幂等事务结算和管理员累计充值额度展示。
- 批量测活与并发防护：请求入口顺序、Redis / 内存计数、临时限制、可信代理边界、分组白名单和管理后台配置页。
- 中文化维护：后端 `zh-CN` locale、default 前端中文 locale、classic 前端中文显示与语言切换移除状态。
- 自定义徽标：default/classic `public/custom-logo.png`、Logo 站内路径校验和 favicon 同步逻辑。
- 前端构建：classic `rsbuild.config.ts` 的 Semi / `date-fns` 解析逻辑，以及 default 额度单位展示。
