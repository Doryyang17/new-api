# New API 二开版本

本仓库是基于 [QuantumNous/new-api](https://github.com/QuantumNous/new-api) 的 fork 二开版本，用于在原有 New API 能力基础上维护自定义功能。

## Fork 基线

- Fork 仓库：[Doryyang17/new-api](https://github.com/Doryyang17/new-api)
- 上游仓库：[QuantumNous/new-api](https://github.com/QuantumNous/new-api)
- 当前二开基线：`12fc0100`
- 基线提交说明：`Bump Electron lockfile dependencies`
- 基线版本描述：`v1.0.0-rc.15-22-g12fc0100`

## 定制化功能

### 系统可用时间

新增「系统可用时间」功能，用于配置服务不可用时间段。当前默认目标场景为 `22:00-08:00` 宵禁窗口。

功能行为：

- 管理后台新增「定制化功能」菜单，并在该菜单下提供「系统可用时间」配置。
- 支持配置是否启用、不可用开始时间、不可用结束时间、判定时区和提示文案。
- 宵禁状态下，主页展示替换为宵禁提示页，导航栏和站内其他页面仍保持可访问。
- 宵禁状态下，模型调用类 API 直接返回 `503` 和 `system_curfew` 错误，不再进入渠道分发，也不会调用上游渠道 API。
- 宵禁拒绝会写入错误日志，保留请求路径、时间窗口、时区、Token 上下文等审计信息。
- 同步覆盖新版前端和经典前端，并补充浅色、深色主题下的主页展示。

## 保留能力

除上述定制化功能外，项目继续继承 New API 原有能力，包括多模型网关、渠道管理、用户与令牌管理、用量日志、计费配置、模型广场、排行榜和多语言前端等功能。

## 上游同步说明

本仓库后续二开应继续以 `upstream` 作为上游来源，以 `origin` 作为个人 fork 发布目标。合并上游更新时，应先确认定制化功能没有被覆盖，并重点检查系统可用时间的后端强制拦截、前端主页替换和管理后台配置入口。
